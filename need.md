# Meituan AI-Diet 需求文档（当前版本）

## 1. 产品目标

构建一个面向真实用户的外卖对话助手，让用户通过自然语言完成“表达需求 -> 获取推荐 -> 持续追问 -> 收敛选择”。

核心目标：

- 降低点餐决策成本
- 支持多轮对话中的条件调整
- 结果可解释、可比较、可追问

## 2. 用户核心场景

- 预算+口味+时效快速筛选（如“预算30以内，清淡不油腻，送达快一点”）
- 追问微调（如“换个更近的”“再便宜一点”）
- 解释型追问（如“为什么推荐这家”“这家健康吗”）
- 对比决策（如“对比前两家”）

## 3. 当前功能边界

- 多轮对话：支持 `recommend / qa / smalltalk / reset`
- 结构化记忆：预算、距离、时效、口味偏好
- 作用域重排：追问优先在上轮候选内优化
- 硬约束执行：`换个更近的` 可执行最近约束
- 实时位置：可接入高德 POI 周边检索
- 菜品来源：provider / crawler / llm infer / template 多级回退

## 4. 非功能需求

- 稳定性：外部服务异常时必须可回退
- 可观测性：输出 debug（agent steps、耗时、fallback）
- 可回归性：对话边界有自动回归脚本覆盖
- 可扩展性：多-agent分层，支持后续新增能力

## 5. 技术选型（已落地）

- 后端：FastAPI
- 前端：Streamlit
- 多-Agent：Orchestrator / Retrieval / Response / Memory
- 向量语义：Embedding + cosine（可开关）
- 地图服务：高德 Web API
- 存储：
  - 用户偏好：JSON
  - 对话会话：SQLite

## 6. 验收指标（当前迭代）

- 对话路由正确率：QA/推荐/闲聊/重置不串线
- 追问有效率：`换更近`能触发实际变化或明确无解提示
- 响应时延：追问 fast_mode 体感明显快于首轮
- 可解释性：支持 top1解释、健康回答、前两家对比卡片
